# Welcome to MAD-I Handbook

## What is this course about?

Imagine you want to open a **restaurant**. You need:
- A **building** to put it in (the structure)
- A **kitchen** to cook food (the backend logic)
- A **menu and nice plates** to serve customers (the frontend)
- A **way for customers to order** (the interface/API)

**Modern Application Development (MAD)** teaches you how to build the *digital version* of that restaurant — a web application — from scratch.

This course is like a full recipe book. By the end, you will know how the Internet works, how to build web pages, how to make your server remember users, and how to deploy your app so the whole world can use it.

```

| 2 | WEB → How does the Internet work? |
```

Think of this as building a house. You start with the foundation, then the walls, then the roof, and finally the furniture.

## The Golden Rule of This Course

[TIP]
**Separation of Concerns**: Keep your code organized by making sure each part of your program does **one job** and one job only. The code that talks to the database should NOT be the same code that decides how buttons look. This is the single most important idea in software engineering.
[/CALLOUT]

## Glossary of Key Ideas

| Term | Simple Meaning |
| :--- | :--- |
| **App** | A software program that does a specific job |
| **Client** | The device (browser/phone) that asks for something |
| **Server** | The machine that provides the answer |
| **Protocol** | A set of agreed-upon rules for communication |
| **Framework** | A pre-built toolkit that makes common tasks easier |
