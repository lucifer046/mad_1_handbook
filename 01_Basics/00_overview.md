# MAD Handbook Overview

Modern Application Development (MAD) is a multi-disciplinary field that combines software architecture, web technologies, backend logic, and DevOps practices to build scalable, resilient, and user-centric applications. This handbook serves as a comprehensive guide to the core principles and technologies covered in the MAD course.

[NOTE]
This handbook is designed as a data-driven SPA. You can search for any topic using **Ctrl+K** or by browsing the categories in the command palette.
[/CALLOUT]

## Course Philosophy
The course focuses on the **Web as a Platform**. By understanding the constraints and capabilities of the web, we can apply sound engineering principles like **Separation of Concerns** and the **MVC Pattern** to build applications that work across devices and environments.

## Topic Roadmap
1.  **Apps and Architectures**: Understanding the building blocks of software.
2.  **Web Fundamentals**: The history and mechanics of the Internet (TCP/IP, HTTP).
3.  **Information Representation**: How data is encoded (ASCII, Unicode, UTF-8).
4.  **Frontend Core**: HTML5, CSS3, and the Document Object Model (DOM).
5.  **Backend Logic**: MVC, Flask, Routing, and Templates.
6.  **Data Persistence**: SQL, NoSQL, and Database Design.
7.  **Security**: Authentication, Authorization, and Common Vulnerabilities.
8.  **Operations**: Testing, Deployment, and CI/CD pipelines.

## Key Takeaways
- MAD is about building robust client-server systems.
- The web is the most versatile platform for modern apps.
- Separation of concerns is the "Golden Rule" of software architecture.
