# Beyond HTML

The web is constantly evolving. What started as a simple document sharing system has become a powerful application platform.

## Evolution of Markup
-   **SGML**: The complex ancestor of HTML and XML.
-   **XML**: A strict, structured data format.
-   **XHTML**: An attempt to make HTML follow strict XML rules.
-   **HTML5**: The modern living standard. It breaks from strict SGML/XML rules to prioritize real-world browser behavior.

## Web Components
Web Components allow you to create reusable custom elements with their own functionality and styling, encapsulated from the rest of the page.
1.  **Custom Elements**: Defining new HTML tags.
2.  **Shadow DOM**: Keeping component styles private.
3.  **HTML Templates**: Defining markup that isn't rendered until needed.

## Modern Frameworks
Frameworks help manage the complexity of large scale applications.
-   **Backend**: Flask, Django, Ruby on Rails.
-   **Frontend**: React, Vue, Svelte.

### Why React?
React is a declarative, component-based library for building user interfaces.
-   **Declarative**: You describe *what* you want the UI to look like, and React handles the *how*.
-   **Components**: Break the UI into small, reusable pieces.
-   **Virtual DOM**: Optimizes updates to the real DOM for better performance.

## Glossary
- **Framework**: A set of tools and rules that simplify software development.
- **Component**: A self-contained, reusable piece of a UI.
- **Shadow DOM**: A web standard that provides encapsulation for JS and CSS in Web Components.
