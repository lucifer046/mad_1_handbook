# Accessibility and UI Design

Designing for the web means designing for **everyone**. This includes users with visual, auditory, motor, or cognitive impairments.

## Accessibility (a11y)
Follow the **POUR** principles defined by W3C:
1.  **Perceivable**: Users must be able to perceive the info (e.g., alt text for images).
2.  **Operable**: Interface cannot require interactions a user cannot perform (e.g., keyboard-only navigation).
3.  **Understandable**: Users must be able to understand the info and operation (e.g., clear error messages).
4.  **Robust**: Content must be compatible with a wide variety of user agents, including assistive technologies.

[NOTE]
**ARIA Attributes**: Use `aria-` attributes (like `aria-label`) to provide extra context to screen readers that isn't visually obvious.
[/CALLOUT]

## UI Design Heuristics
Jakob Nielsen's **10 Heuristics** for User Interface Design are a set of general principles for interaction design:
1.  Visibility of system status.
2.  Match between system and real world.
3.  User control and freedom.
4.  Consistency and standards.
5.  Error prevention.
6.  Recognition rather than recall.
7.  Flexibility and efficiency of use.
8.  Aesthetic and minimalist design.
9.  Help users recognize, diagnose, and recover from errors.
10. Help and documentation.

## Wireframing
A **Wireframe** is a low-fidelity visual guide that represents the skeletal framework of a website. It focuses on space allocation, prioritization of content, and available functionality.

[TIP]
**Don't use real data in wireframes**. Use "Lorem Ipsum" placeholder text and gray boxes for images to avoid distracting stakeholders with visual details like colors and fonts.
[/CALLOUT]

## Glossary
- **WCAG**: Web Content Accessibility Guidelines.
- **POUR**: Perceivable, Operable, Understandable, Robust.
- **Heuristic**: A broad rule of thumb or best practice.
